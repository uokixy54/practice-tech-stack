# 学習メモ
## サーバーサイドとは
- サーバー上で実行される処理
### ex) データベース、外部のシステム...


## クライアントサイドとは
- ユーザが見ている画面
### ex) アプリ、音声認識デバイス、IoT、ブラウザ...


## レンダリングとは
- htmlにcssやjavascriptを当てて人間がみやすいようにする


## AJAX(Asynchronous JavaScript and XML)
- ページを遷移しない状態でユーザインタフェースを構築する仕組み
### ex) GoogleMaps


## webサイトがどうやってみられるか（javascript未使用）
1. ユーザがURL(example.com)を検索する
2. ブラウザがURLを受け取ってサーバーに**リクエスト(req)**をする
3. ブラウザからの要求によって**レスポンス(res)**を返す
4. ブラウザのレンダリングを行う
5. ユーザがwebサイトを閲覧
### **ページを遷移するたびにサイレンダリングされる**


## webサイトがどうやってみられるか（javascript使用）
1. ユーザがURL(example.com)を検索する
2. ブラウザがURLを受け取ってサーバーに**リクエスト(req)**をする
3. ブラウザからの要求によって**レスポンス(res)**を返す
4. ブラウザのレンダリングを行う
5. ユーザがwebサイトを閲覧
6. ほかのURL(example.com/about)を検索
7. サーバー側からはJSONが帰ってくる
8. ページ全体が更新かかることはなユーザが閲覧できる


## まとめ
### Javascriptを用いることによって
- より体験が良いWebサイトが見れる
- レスポンスで返すのはJSON(初回はHTML)
- ページ遷移時に最小限のレンダリング
